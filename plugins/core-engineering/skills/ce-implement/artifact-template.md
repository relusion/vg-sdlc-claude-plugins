# Feature-Implement — Artifact Template (`verification.md`)

Write-time template for the `implement` skill (orchestrator: `SKILL.md`). Load it at the Stage 3 **Accept** step; do not reconstruct the format from memory. It records, per the spec's acceptance criteria, what was verified and how — the existence flag every downstream gate (`/ce-verify`, `/ce-ship-release`, auto-build) reads to call the feature `implemented`.

---

````markdown
# Verification: <plan-slug>/<feature-id>

> Generated by `/ce-implement`
> Feature: <id> · <title>   ·   Status: implemented | partial
> Last updated: YYYY-MM-DD

## Acceptance Criteria

Each AC traces to its test cases and their results — the AC → TC → result chain
(`auto` green, `manual` verdict). No criterion is `pass` without a result.

| AC | Test cases | Modality · Verification | Result | Evidence |
|---|---|---|---|---|
| AC-1 | TC-1, TC-2 | http · auto | pass | <test name / output ref> |
| AC-2 | TC-3 | browser · manual:harness-gap | pass | <screenshot / who / when> |
| AC-3 | TC-4 | manual · manual:judgment | deferred | <evidence captured; verdict deferred to the human> |

- **`auto`** — the test result is the evidence (name the test / suite output).
- **`manual:harness-gap`** — an evidence-backed verdict: the script run + who + when.
- **`manual:judgment`** — captured evidence with the verdict **deferred** to the human; never self-certified.

## Test Run

- Suite: `<command>` → <N passed / M total>
- Build: `<command>` → pass / fail
- Lint / type-check: `<command>` → pass / fail
- Interface conformance *(when the feature exposes a foundationed surface)*: `<checker command>` → pass / fail

## Files Changed

- `path/to/file` — <what changed>

## Try It Yourself

The grounded runbook — the **real commands run in Stage 2**, not narrated guesses —
shaped by feature type:

- **user-facing / CLI** — start command, entry point or URL, the journey to walk.
- **API / integration** — the real requests (e.g. `curl`) and a correct response.
- **SDK / library** — a minimal usage snippet.
- **foundation / infrastructure** — the command that exercises it + the success signal.

*(For a mid-feature resumable stop — not all tasks `done` — set Status: partial and label this section partial.)*
````
