# Feature-Review — Artifact Templates

Write-time templates for the `review` skill (orchestrator: `SKILL.md`). Load them at the Stage 2 write step; do not reconstruct any format from memory. Holds the `code-review.md` findings file, the `review-summary.json` machine-summary schema, the `review-policy.md` / `review-learnings.md` calibration-and-memory templates, and the closing run-summary block.

---

````markdown
| Performance | … | … | … | … |

## CR-N — <short title>  [<severity>· <confidence (High only)>]

- **Location:** `path/to/file:line`
- **Observation:** <what the reviewer saw>
- **Evidence:** ```<snippet>``` (or `evidence/CR-N.txt`)
- **Suggested escalation:** <skill>

  ↳ Escalated to: <skill> | Deferred as: <note> | Dismissed: <reason>

## Triaged

| ID | Lens | Severity | Confidence | Triage | Escalation | Date |
|---|---|---|---|---|---|---|
| CR-1 | security | high | confirmed | Escalate | /core-engineering:ce-implement 04-… | 2026-MM-DD |

## Previously dismissed (suppressed this run)

Findings the lens walk re-surfaced but a prior human `Dismiss` suppressed (via `review-learnings.md`). Recorded, not raised; never block. Listed so suppression is visible, never silent — promote a wrong suppression back by editing the cited `RL-N`.

| Finding shape | Lens | Matched rule | Originally dismissed |
|---|---|---|---|
| <short> | maintainability | RL-2 (`src/generated/**`) | <plan>/03-… · 2026-MM-DD · "generated code, not reviewed" |
````

Only a **confirmed** High in a behavioral lens blocks under auto-build; a **suspected** High is recorded here and surfaced at the end-review; a **suppressed** finding is shown above and never blocks.

## Machine Summary — `review-summary.json`

The current-state, machine-readable gate input — **per feature**, at `docs/plans/<slug>/specs/<id>/review-summary.json`, **overwritten** each review pass (peer of `code-review.md` and `verification.md`). It exists so the auto-build orchestrator and any CI gate read the block decision from **one precomputed key** (`blocking_high`) instead of parsing prose. `code-review.md` keeps the cumulative human record; this keeps the current machine state.

```json
{
  "status": "blocked",
  "feature_id": "04-checkout",
  "plan_slug": "<slug>",
  "mode": "blocking-on-high",
  "reviewed_at": "YYYY-MM-DD",
  "findings_total": 7,
  "blocking_high": 1,
  "suppressed": 2,
  "by_severity": {
    "high":   { "total": 2, "confirmed": 1, "suspected": 1 },
    "medium": { "total": 3 },
    "low":    { "total": 2 }
  },
  "by_lens": {
    "correctness": 1, "security": 2, "performance": 1,
    "maintainability": 2, "conformance": 1, "simplicity": 0
  },
  "findings": [
    {
      "id": "CR-1", "lens": "security", "severity": "high",
      "confidence": "confirmed", "reproduced": true,
      "file": "src/orders.py:88",
      "observation": "<short>", "suggested_escalation": "/core-engineering:ce-implement"
    },
    {
      "id": "CR-2", "lens": "correctness", "severity": "high",
      "confidence": "suspected", "reproduced": false,
      "file": "src/cart.ts:42",
      "observation": "possible null deref — path not shown reachable", "suggested_escalation": "/core-engineering:ce-implement"
    }
  ]
}
```

- **`status`** — `"pass"`, or `"blocked"` iff `blocking_high > 0`. It reflects the
  machine gate state. Optional **`mode`** is provenance only; the deterministic
  gate intentionally acts only on `status` and `blocking_high`.
- **`blocking_high`** is the single precomputed gate count: the number of findings where `severity == "high" AND confidence == "confirmed" AND lens ∈ {correctness, security}`. The deterministic gate reads that count, requires a non-negative integer, and checks that `status` agrees with zero/non-zero. It never re-derives the predicate from the finding list. (Performance is excluded for free — it can never be High.)
- **`by_severity.high`** carries the `confirmed` / `suspected` split (the only severity that is confidence-tagged); `medium` / `low` carry just `total` (untagged — so `total` need not equal a confidence sum). **`reproduced`** is `true|false` for a behavioral High, `null` for a judgment High. Field names are `snake_case`; severity keys are loudest-first; the six `by_lens` keys are the fixed set.
- **`suppressed`** is the count of findings the lens walk re-surfaced this run but a prior dismissal (a `review-learnings.md` rule) suppressed — recorded under "Previously dismissed", excluded from `findings_total` and `blocking_high`. It is here so the end-review and `/core-engineering:ce-retro` see suppression volume mechanically (a climbing number is a promote-to-`review-policy.md` signal).

## Calibration & Memory Templates

Two human-facing artifacts give review a memory so it stops starting cold. **`review-policy.md`** is repo-level and **hand-authored** (never written by this workflow); **`review-learnings.md`** is per-plan and **append-only** (this workflow appends one entry per human `Dismiss`).

**`docs/plans/review-policy.md`** (repo-level, human-owned — sibling of `vc-policy.md`):

````markdown
# Review Policy

## Severity calibration — what counts as High HERE
| Lens | This repo's bar |
|---|---|
| security | injection / authz / secret-in-log always High |
| correctness | unhandled-error-path High only on a public entrypoint |
| maintainability | duplication under 3 sites is not a finding |

## Nit caps
- Max Low / style findings reported per feature: 5 (overflow batched as "plus N more nits")
- Naming / formatting deferred to the linter: yes

## Skip-paths (generated / vendored — excluded from the lens walk)
- `src/generated/**`
- `**/*.min.js`
- `vendor/**`

## Re-review convergence
- After a feature's first review, report: High only
````

**`docs/plans/<slug>/review-learnings.md`** (per-plan, append-only — written on a human `Dismiss`):

````markdown
# Review Learnings: <plan-slug>

> Append-only. One entry per human `Dismiss` at /core-engineering:ce-review triage or /core-engineering:ce-auto-build end-review.
> SUPPRESSES dismissed false-positive shapes. (Real recurring HAZARDS go to docs/plans/patterns.md — opposite intent.)
> Committed (shared team + CI memory). Match is by shape, not file:line.

## Dismissed (suppression rules)

### RL-1 — <short title>
- lens: maintainability
- pattern: <normalized, location-independent finding shape>
- path_glob: src/generated/**        # optional scope
- severity_ceiling: low              # suppress only at/below this severity (default: the dismissed finding's severity)
- reason: <human's dismissal rationale, verbatim from triage>
- origin: <plan-slug>/04-checkout · CR-3 · 2026-MM-DD
````

A recurring `RL-N` (the same shape dismissed across features / plans — surfaced by `/core-engineering:ce-retro`) is the signal to **promote** it by hand into `review-policy.md`'s skip-paths or nit caps — the repo-wide generalization the human owns.
